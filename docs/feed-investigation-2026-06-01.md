### Root Cause

**The `you_are_playing` query only ever matches `scrapedDate = today (UTC)`.** Once a session day passes midnight UTC, it is permanently excluded from the live query. There is no retroactive mechanism to create a "I played" item after the fact.

The feed is entirely automatic — no user action required — but the `you_are_playing` logic has a hard expiry tied to the current UTC date, not the session's end time in Vietnam.

#### Sequence of events

- **Profile created:** 2026-05-24
- **First-ever feed API call for this profile:** 2026-05-29 06:33 UTC (13:33 VN) — confirmed by `created_at` of the oldest feed rows
- **May 27 session (99525):** By May 29, `scrapedDate='2026-05-27'` is already in the past. The `you_are_playing` query hard-codes `scrapedDate = todayStr ('2026-05-29')`, so session 99525 is **never evaluated** — no feed item is ever created.
- **May 29 session (111005):** The feed ran at 06:59 UTC (13:59 VN) on May 29 — 4 minutes after the session started (13:55 VN). `scrapedDate = '2026-05-29' = todayStr` ✓ → `you_are_playing_111005` persisted at 06:59 UTC.

## Action Items / Recommendations

### Bug — `you_are_playing` is permanently lost for past sessions (CONFIRMED)

The `you_are_playing` query in `/api/feed/route.ts` is:
```ts
session: {
  scrapedDate: todayStr,          // ← only today's date
  startTime: { lte: nowTimeVN },
}
```
This means any session from a previous day can never be captured or backfilled.

**Fix proposal (2 options):**

**Option A — Extend the live query** (minimal change)  
Change the `you_are_playing` query to also look back N days (e.g. 5) using the same `fiveDaysAgo` cutoff already used for `played`, but filter to sessions where `endTime` has already passed. Return these as a new type `played_self` so the UI label is past-tense.

**Option B — Scraper-side backfill** (more robust)  
In the scraper's daily run, for every active Reclub user with a profile, upsert a `played_self` feed item for each session they appeared on in the last 7 days. Same pattern as how `played` items are created for followees — just apply it to `userId = viewer.reclubUserId`.

**Recommendation:** Option A is faster and keeps everything in the API route. The change is about 5 lines.

### Also — Stale `you_are_playing` label in UI

The item `you_are_playing_111005` in feed history still says "You are playing" (present tense) in the `FeedItemRow` component. A simple fix: if `item.timestamp` is more than 3 hours ago, render it as "You played" instead.
